# CONCLUSIONS AND RECOMMENDATIONS

This chapter presents the conclusions and recommendations of the study.

## Conclusions

The study successfully addressed the problem of inefficient manual document request processes at the Computer Arts and Technological College (CATC) by developing a web-based document request system using the Django framework. The prototype was fully implemented and verified, demonstrating the effective application of the Model-View-Controller (MVC) architectural pattern, layered architecture, and secure authentication mechanisms such as OTP-based login. The system automates student document requests, approval workflows for registrars, and payment confirmations for cashiers, significantly reducing processing time and errors compared to traditional manual methods. Through rigorous testing and deployment on a local server with SQLite database, the prototype proved functional, user-friendly, and scalable for small-scale institutional use, confirming that the defined objectives were met. This reflects critical thinking in evaluating the system's ability to handle real-world scenarios, such as concurrent user access and data integrity, while adhering to best practices in web development.

## Recommendations

Reflecting on the development process, the choice of Django as the web framework was highly appropriate due to its built-in security features, ORM for database abstraction, and rapid development capabilities, which streamlined the implementation of complex features like role-based access control and email notifications. The layered architecture ensured maintainability, and the use of SQLite for prototyping was suitable, though transitioning to PostgreSQL is advised for production scalability. Future research should explore integrating advanced features such as real-time notifications via WebSockets, mobile app compatibility, and AI-driven document verification to enhance user experience and security. Additionally, conducting user acceptance testing with actual CATC stakeholders would validate the system's effectiveness in a live environment, while investigating cloud deployment options like AWS or Heroku could address high-traffic demands. Overall, the methods employed—agile development with iterative testing—proved effective, and continued refinement based on user feedback is recommended to ensure long-term relevance and adaptability.
